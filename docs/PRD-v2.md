# BeanTracker PRD v2

## 1. 문서 목적

이 문서는 BeanTracker의 제품 목표, 사용자 가치, 출시 범위, 핵심 기능, 계산 규칙을 확정하여 실제 개발과 구현 우선순위 판단의 기준으로 사용한다.

## 2. 제품 한 줄 정의

BeanTracker는 홈카페 사용자가 원두 소비를 최소 입력으로 기록하고, 월별 절약액을 직관적으로 확인하게 해주는 개인용 대시보드 앱이다.

## 3. 문제 정의

홈카페를 즐기는 사용자는 집에서 커피를 내려 마실 때 실제로 얼마나 절약하고 있는지 체감하기 어렵다.  
기록 앱은 많지만, 절약 체감과 소비 관리라는 핵심 동기를 동시에 만족시키지 못하거나 입력 피로가 커서 습관화되기 어렵다.

## 4. 제품 목표

### 4.1 사용자 목표

- 원두 소비량을 쉽게 기록한다.
- 이번 달 절약액을 한눈에 확인한다.
- 기록 직후 절약액이 올라가는 보상을 경험한다.
- 월말에 월별 소비와 지출을 간단히 리뷰한다.

### 4.2 제품 목표

- App Store에 배포 가능한 실제 앱을 만든다.
- 기록 마찰이 낮아 개인이 지속적으로 사용할 수 있어야 한다.
- 포트폴리오 관점에서 모듈 아키텍처, Preview 친화 구조, TCA 설계를 설득력 있게 보여준다.

## 5. 핵심 사용자

### 5.1 1차 타깃

- 홈카페를 꾸준히 하는 1인 사용자
- 원두를 직접 구매해 소비량을 관리하고 싶은 사람
- 절약 금액을 확인하며 루틴의 성취감을 느끼고 싶은 사람

### 5.2 사용자의 실제 동기

- 원두를 얼마나 쓰고 있는지 알고 싶다.
- 카페 대신 집에서 마시며 얼마나 아끼는지 확인하고 싶다.
- 복잡한 기록 없이 빠르게 남기고 싶다.

## 6. 제품 원칙

- 절약액이 제품의 대표 지표여야 한다.
- 기록은 상세한 커피 저널보다 빨라야 한다.
- 입력보다 결과 확인의 보상이 커야 한다.
- 계산은 회계 수준의 정밀함보다 사용자가 납득할 수 있는 일관성이 중요하다.
- 구조는 기능 구현보다 먼저 Preview 속도와 유지보수성을 해치지 않아야 한다.

## 7. 성공 기준

### 7.1 사용자 성공 기준

- 원두 등록과 추출 기록을 몇 초 안에 끝낼 수 있다.
- 앱을 열자마자 이번 달 절약액을 이해할 수 있다.
- 추출 직후 절약액 증가 애니메이션이 명확한 보상으로 작동한다.

### 7.2 제품 성공 기준

- iPhone, iPad, macOS에서 핵심 기능이 동일하게 동작한다.
- 한국어와 영어를 모두 지원한다.
- 같은 Apple ID 환경에서 데이터 동기화가 가능하다.
- 다른 홈카페 사용자에게 보여줬을 때 써보고 싶다는 반응을 이끌 수 있다.

## 8. 핵심 사용자 플로우

### 8.1 최초 실행

1. 사용자의 기준 카페 커피 가격을 입력받는다.
2. 값은 `UserPreference.standardCafePrice`에 저장한다.
3. 등록된 원두가 없으면 원두 등록으로 이동을 유도한다.

### 8.2 원두 등록

1. 사용자가 원두명, 로스터리, 총중량, 가격, 구매일을 입력한다.
2. 새 원두는 기본적으로 활성 상태로 저장된다.
3. 등록이 끝나면 추출 기록 화면으로 자연스럽게 이동할 수 있어야 한다.

### 8.3 추출 기록

1. 활성 원두 중 가장 최근에 사용한 원두가 기본 선택된다.
2. `usedWeight`는 마지막 추출값을 기본값으로 채운다.
3. 사용자는 원두와 그람 수만 확인한 뒤 기록을 완료한다.
4. 완료 즉시 대시보드 수치가 갱신된다.
5. 이번 달 절약액이 애니메이션으로 증가한다.

### 8.4 월말 리뷰

1. 사용자는 대시보드를 열어 이번 달 절약액을 먼저 본다.
2. 월별 바 차트로 이전 달과 흐름을 확인한다.
3. 월별 상세 통계에서 원두 사용량과 구매 비용을 확인한다.

## 9. 정보 구조

- 탭 1: 대시보드
- 탭 2: 커피 내리기
- 탭 3: 원두 창고
- 보조 화면: 온보딩, 설정

## 10. 기능 요구사항

### 10.1 대시보드

대시보드는 제품의 핵심 화면이며 절약 체감이 가장 강하게 드러나야 한다.

필수 요소:

- Hero 지표로 `이번 달 절약액` 표시
- 추출 기록 직후 절약액 증가 애니메이션
- 월 전환 UI
- 월별 절약액 바 차트
- 월 상세 통계
  - 이번 달 절약액
  - 이번 달 원두 소비량(g)
  - 이번 달 원두 구매 비용
- 현재 원두 상태 카드
  - 남은 원두량(g)
  - 예상 남은 잔 수
  - 소진 예상 여부

동작 규칙:

- 활성 원두가 여러 개면 최근 추출에 사용한 활성 원두를 현재 원두로 본다.
- 추출 이력이 없으면 가장 최근에 등록한 활성 원두를 현재 원두로 사용한다.
- 활성 원두가 없으면 빈 상태와 원두 등록 유도 화면을 보여준다.

### 10.2 커피 내리기

추출 기록 화면은 가장 빠르게 기록을 완료할 수 있어야 한다.

입력 항목:

- 원두 선택
- 사용량(g)

동작 규칙:

- 1회 추출은 1잔으로 간주한다.
- 2잔 이상 추출은 1.0 범위에서 제외한다.
- 기본 원두는 가장 최근에 사용한 활성 원두다.
- 기본 사용량은 마지막으로 입력한 `usedWeight`다.
- 추출 기록은 생성 후 수정할 수 없다.
- 추출 기록은 삭제할 수 있다.

### 10.3 원두 창고

원두 창고는 구매 이력과 현재 재고 상태를 관리한다.

필수 요소:

- 새 원두 등록 폼
- 활성 원두 목록
- 소진 원두 목록
- 원두 상세 정보 표시
- 잘못 등록한 원두 삭제
- 수동 소진 전환

동작 규칙:

- 원두는 생성 후 수정할 수 없다.
- 원두는 삭제할 수 있다.
- 활성 원두는 여러 개 허용한다.
- 누적 사용량이 총중량 이상이면 `소진 예상` 상태를 표시한다.
- 최종 소진 여부는 사용자가 직접 전환한다.

### 10.4 온보딩

필수 요소:

- 기준 카페 가격 입력
- 빈 원두 창고 안내
- 첫 원두 등록 유도
- 첫 추출 기록 유도

### 10.5 설정

필수 요소:

- 기준 카페 가격 변경
- 언어 또는 시스템 언어 반영 진입점

### 10.6 위젯

위젯은 1.0에 포함하되, 포트폴리오 강화를 위한 범위로 관리한다.

필수 요소:

- 소형 위젯
- 앱의 추출 기록 화면으로 바로 진입하는 빠른 기록 버튼

1.0 제외:

- 위젯 내부에서 월 통계 상세 표시
- 위젯에서 직접 기록 완료

## 11. 계산 및 비즈니스 규칙

### 11.1 핵심 공식

- 원두 g당 원가 = `Bean.price / Bean.totalWeight`
- 1잔 원가 = `원두 g당 원가 * BrewLog.usedWeight`
- 1잔 절약액 = `UserPreference.standardCafePrice - 1잔 원가`
- 이번 달 절약액 = 선택한 월의 `1잔 절약액` 합계
- 이번 달 원두 소비량 = 선택한 월의 `BrewLog.usedWeight` 합계
- 이번 달 원두 구매 비용 = 선택한 월에 구매한 `Bean.price` 합계
- 남은 원두량 = `Bean.totalWeight - 해당 원두의 누적 usedWeight`
- 예상 남은 잔 수 = `남은 원두량 / 마지막 usedWeight`

### 11.2 해석 규칙

- 이번 달 절약액은 현금 흐름 기준이 아니라 소비 기준이다.
- 월 구매 비용은 별도로 노출해 사용자가 지출과 절약을 함께 이해할 수 있게 한다.
- 고가 원두를 사용할 경우 절약액은 음수가 될 수 있다.
- 절약액과 구매 비용은 서로 다른 의미의 지표이므로 UI에서 명확히 구분해야 한다.

## 15. 리스크와 대응

### 리스크 1. 기록이 귀찮아서 사용이 끊길 수 있음

대응:

- 최근 원두 기본 선택
- 마지막 사용량 자동 채움
- 빠른 기록 위젯 제공
- 기록 후 즉시 보상 애니메이션 제공

### 리스크 2. 절약액이 직관적으로 납득되지 않을 수 있음

대응:

- 계산 규칙을 단순하게 유지
- 월 구매 비용을 별도 표시
- 지표 의미를 UI와 문서에서 명확히 분리

### 리스크 3. 활성 원두가 여러 개일 때 현재 상태가 모호해질 수 있음

대응:

- 대시보드는 최근 사용 원두를 기준으로 상태 카드 구성
- 원두 창고에서는 전체 활성 목록을 명확히 노출

## 16. 수용 기준

- 사용자는 앱 최초 실행 후 기준 카페 가격을 입력할 수 있어야 한다.
- 사용자는 원두를 등록하고 활성 목록에서 확인할 수 있어야 한다.
- 사용자는 기본값이 채워진 상태로 추출을 빠르게 기록할 수 있어야 한다.
- 추출 완료 직후 대시보드 절약액이 갱신되고 애니메이션이 실행되어야 한다.
- 사용자는 월별 차트와 월 상세 통계를 통해 월별 흐름을 확인할 수 있어야 한다.
- 사용자는 원두와 추출 기록을 삭제할 수 있어야 한다.
- 같은 Apple ID 환경에서 데이터가 여러 기기 간 동기화되어야 한다.

## 17. 제품 정의 요약

BeanTracker의 본질은 원두 관리 앱이 아니라, 홈카페 절약을 월 단위로 시각화하는 저마찰 기록 시스템이다.
